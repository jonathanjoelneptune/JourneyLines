# GlobeHoppers v8.2 Work Package 3

## Scope

- Create-only cloud trip writes through `create_private_trip`.
- Atomic creation of locations, trip, ordered legs, and Hopper assignments.
- Edit, delete, batch add, and timeline reordering remain disabled.

## Acceptance

- User A can add and reload a private trip.
- User B cannot see User A trip.
- Multi-leg and round-trip creation persist in order.
- Invalid cross-map Hopper assignments fail.
- Failed transactions leave no partial trip.
