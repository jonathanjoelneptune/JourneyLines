# GlobeHoppers v8.2.5 — Work Package 4 Concurrency Feedback Hotfix

## Purpose

Ensure a stale Edit Hop save never leaves the Save button stuck on `Saving…`.

## Expected conflict behavior

1. The database rejects the stale revision.
2. The editor remains open.
3. The Save button returns to `Save Hop`.
4. The editor displays and alerts: `This trip changed in another session. Reload before saving.`
5. The newer database version is not overwritten.

## Timeout behavior

Cloud update requests are bounded to 30 seconds. A network or unresolved request releases the busy state and presents a retryable error.
