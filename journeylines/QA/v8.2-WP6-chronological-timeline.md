# GlobeHoppers v8.2.8 Work Package 6

## Chronological Timeline and Cloud-Only Account Integrity

- Trip order is determined by start date, end date, creation timestamp, and UUID.
- Manual timeline ordering is not available.
- `sort_order` remains a compatibility field but does not drive cloud playback.
- Signed-in cloud failures display a retryable error and never fall back to bundled personal data.
- Trip leg order continues to use `trip_legs.leg_order`.
- Migration 013 removes the temporary `timeline_order_revision` compatibility column after Preview verification.
